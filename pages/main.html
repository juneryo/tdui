后续会提供各种页面模板供参考。

  • 升降框

  • 日历

依赖

bootstrap 及 bootstrap-material-design
样式依赖于 bootstrap v3.3.6 以及 bootstrap-material-design v0.5.7 ，渲染material风格的UI。同样，也支持原始bootstrap风格。
avalon
控制依赖于简单易用的mvvm框架 avalon v1.5.x ，构建专属的td标签库。

如何使用

TDUI通过使用avalon构建私有td标签库，因此请先了解如何使用 avalon。

以 panel 标签为例：

<td:panel $id="test_panel" config="$opt" title="标题">
  <div slot="content">
    //其他html片段
  <div/>
</td:panel>

var vpanel = avalon.define({
  $id: 'test',
  $opt: {
    buttons: [{
      display: '按钮', fun: function(ev, vm) {
        vm.setTitle('xxx');
      }
    }]
  }
});
UI组件的标签属性：
直接配置在标签中的title为标签属性，主要配置一些简单类型的参数。
UI组件的配置参数：
标签中的config对应的$opt对象为模型配置对象，对象的属性为该标签的配置参数，用于配置方法、对象等不方便直接配置于标签中的参数。
UI组件的插入片段(slot)：
容器类的UI组件都会有相应的插入片段(slot)，可将其他UI组件及html代码插入到指定位置。
UI组件的唯一标识：
$id是avalon为每个组件设置的唯一标识，默认为随机生成，也可以手工进行配置。
UI组件的vmodel：
手工配置$id后，可以通过avalon.vmodels['id值']来获取组件的vmodel。
同样的，组件中某些事件处理方法中的参数vm也是指该组件的vmodel。比如上述例子中的fun: function(ev, vm){...}。
可以直接通过vmodel，调用组件属性或使用组件中提供的方法，请详见各组件API。

其他组件

TD.alert： 测试alert
var main = document.getElementById('idx_main');
var w = TD.css.getSize(main, 'width');
var h = TD.css.getSize(main, 'height');
TD.alert('<strong>主页main尺寸</strong>(宽：' + w + 'px,高：' + h + 'px)');
TD.confirm： 测试confirm
TD.confirm('<strong>这里是confirm</strong>', function() {
    TD.hint('点击了确定', 'success');
}, function() {
    TD.hint('点击了取消');
});
TD.hint： 测试hint
TD.hint('这里是hint', 'info');  //支持success,info,warning,danger这4中类型，默认warning

浏览器支持

支持至IE8及以上IE浏览器，新版本的FireFox、Chrome等。

IE8及以下版本IE对私有标签库的支持需要在head中引入avalon，并将avalon.config紧随其后放置。

<head>
<script type="text/javascript" src="avalon.js"></script>
<script>
    //这个东西必须跟挨着avalon库,并且放到head标签中
    avalon.library("td", {
        $init: function(){},
        $childReady: function(){},
        $ready: function(){},
        $dispose: function(){}
    });
</script>
</head>
